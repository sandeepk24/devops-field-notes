# DevOps Field Notes

Practical field notes from real-world work across:

- AWS EKS
- ECS / containers
- Kubernetes
- CI/CD
- Linux / Unix
- SRE / observability
- AI for DevOps
- CKAD-style patterns
- platform engineering and production operations
- AWS


It is a collection of sharp, practical, operator-focused notes based on real systems, real commands, real debugging, and real platform engineering judgment.

## Structure

- `eks/` - EKS and Kubernetes operational lessons
- `ecs/` - ECS and container runtime notes
- `docker/` - Docker build and image insights
- `kubernetes/` - broader Kubernetes concepts and patterns
- `linux/` - Linux / Unix commands and debugging notes
- `sre/` - reliability, incidents, and service behavior
- `cicd/` - deployment and pipeline insights
- `ai-devops/` - AI use cases and cautions in DevOps
- `observability/` - metrics, logs, traces, alerting
- `ckad/` - hands-on Kubernetes and exam-style patterns
- `field-notes/` - cross-cutting architecture and operational notes
- `aws/` - AWS cloud related concepts 

## Posting style

Each note aims to be:
- short
- practical
- production-aware
- immediately useful
